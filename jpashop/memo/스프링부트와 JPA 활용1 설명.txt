<스프링부트 + JPA>
- 스프링 부트 : 복잡하고 어려운 스프링 기술을 간결하고 쉽게 사용할 수 있도록 도와주는 기술
- JPA : 강력한 자바 ORM 표준 기술
- 위 두가지 기술을 합치면 높은 개발 생산성을 유지하면서 빠르게 웹 어플리케이션을 개발할 수 있다.

1편 - 웹 어플리케이션 개발
1) 프로젝트 환경설정
	SpringBoot, Gradle, Thymeleaf(View Template), JPA, Hibernate, Tomcat
2) 요구사항 분석
	회원기능(회원가입, 회원목록), 상품기능(상품등록, 상품목록), 주문기능(상품주문, 주문내역)
3) 도메인과 테이블 설계
	요구사항 분석을 바탕으로 도메인 모델 설계	
	엔티티 설계
	테이블 설계
	마지막으로 JPA와 엔티티를 ORM으로 매핑한다.
4) 아키텍처 구성
	Controller -> Service -> Repository -> DB
	Controller -> Domain
	Service -> Domain
	Repository -> Domain
	과 같은 아키텍쳐 구성
5) 핵심 비즈니스 로직 개발(회원, 상품, 주문)
	회원, 상품, 주문 도메인 개발
	핵심 비즈니스 로직 개발	
	도메인 주도 설계 이해
6) 테스트
	테스트 케이스 검증
7) 웹 계층 개발
	위에서 했던 것들의 기반위에 웹 계층을 올린다.
	Controller와 Thymeleaf를 올려서 html이 랜더링 되게 만들고, 웹브라우저에서 동작하게 한다.


2편 - API 개발과 성능 최적화
- 등록, 수정, 조회 REST API 개발
- API 개발 실무 노하우
- 성능 최적화
	- JPA 극한의 조회 성능 최적화 노하우 전수
	- 복잡한 예제를 6단계로 성능 튜닝
	- 실무 JPA 성능 문제의 90% 해결
